# 🛡️ Security Policy
Ti ringraziamo per il tuo contributo nel migliorare la sicurezza di questo progetto. La sicurezza è una nostra priorità e apprezziamo sinceramente il tempo e l’impegno dedicati alla segnalazione responsabile di eventuali vulnerabilità. Ti invitiamo a seguire le indicazioni riportate di seguito per consentirci di analizzare e risolvere i problemi nel modo più rapido ed efficace possibile.

## ✅ Stato di supporto

| Versione | Supporto            |
| -------- | ------------------- |
| latest   | ✅  |
| beta     | ⚠️                 |

> La versione di beta è soggetta a bug o vulnerabilità. Il supporto è sempre garantito e si consiglia vivamente di segnalare ogni problematica riscontrata

## 🎟️ Segnalazione di Vulnerabilità

Per segnalare una vulnerabilità di sicurezza, si prega di non aprire issue pubbliche. Utilizzare invece la funzionalità di segnalazione privata di GitHub (“Report a vulnerability”) disponibile nella sezione Security del repository, oppure contattare direttamente i maintainer tramite i canali indicati. Nella segnalazione includere una descrizione dettagliata del problema, i passaggi per riprodurlo, l’impatto potenziale e, se possibile, una proposta di mitigazione. Ci impegniamo a rispondere tempestivamente e a gestire la segnalazione secondo i principi della responsible disclosure.
